## การติดตั้งครั้งแรก

### ความต้องการของระบบ
- มือถือหรือคอมพิวเตอร์ที่มีกล้อง
- Browser ที่รองรับ WebRTC (Chrome, Safari, Edge)
- Internet connection

### ขั้นตอนการติดตั้ง

#### 1. ติดตั้ง Node.js และ Dependencies
```bash
# ตรวจสอบ Node.js version (ต้อง 18+)
node --version

# ติดตั้ง dependencies
npm install
```

#### 2. สร้าง Supabase Project
1. ไปที่ https://supabase.com
2. สร้างบัญชีใหม่ (ใช้ Google Account ได้)
3. คลิก "New Project"
4. ตั้งชื่อ Project: `stu-face-attendance`
5. ตั้ง Database Password (จดไว้)
6. เลือก Region: `Southeast Asia (Singapore)`
7. รอประมาณ 2-3 นาทีจนสร้างเสร็จ

#### 3. ตั้งค่า Database
1. ไปที่ "SQL Editor" ใน Supabase Dashboard
2. คลิก "New query"
3. Copy โค้ดทั้งหมดจาก `supabase/schema.sql`
4. Paste และคลิก "Run"
5. จะเห็นข้อความ "Success" เมื่อสำเร็จ

#### 4. เปิดใช้งาน Realtime
1. ไปที่ "Database" → "Replication"
2. เปิด "Realtime" สำหรับ table `attendance`

#### 5. สร้าง Storage Bucket (ถ้าต้องการเก็บรูป)
1. ไปที่ "Storage"
2. คลิก "New bucket"
3. ชื่อ: `student-photos`
4. เลือก "Public bucket"
5. คลิก "Create bucket"

#### 6. คัดลอก API Keys
1. ไปที่ "Settings" → "API"
2. คัดลอก:
   - `Project URL` → `NEXT_PUBLIC_SUPABASE_URL`
   - `anon public` → `NEXT_PUBLIC_SUPABASE_ANON_KEY`
   - `service_role` → `SUPABASE_SERVICE_ROLE_KEY`

#### 7. ตั้งค่า Environment Variables
```bash
# สร้างไฟล์ .env.local
cp .env.example .env.local

# แก้ไขไฟล์ .env.local ด้วย text editor
# ใส่ค่าที่ copy มาจากขั้นตอนที่ 6
```

#### 8. รันระบบ
```bash
# Development mode
npm run dev

# เปิดเบราว์เซอร์ที่ http://localhost:3000
```

## การตั้งค่า Supabase

### สร้าง Database Schema

ระบบใช้ 4 tables หลัก:

#### 1. service_points (จุดให้บริการ)
- ศูนย์หลัก + หน่วยบริการย่อย 11-15 จุด
- เก็บข้อมูล: ชื่อ, ที่อยู่, สถานะเป็นศูนย์หลักหรือไม่

#### 2. teachers (ครู)
- เก็บข้อมูลครูผู้ใช้งานระบบ
- เชื่อมโยงกับจุดให้บริการ

#### 3. students (นักเรียน)
- เก็บข้อมูลนักเรียนและ embeddings (สูงสุด 20 ชุด)
- embeddings คือข้อมูลใบหน้าที่แปลงเป็นตัวเลข
- rolling update อัตโนมัติ

#### 4. attendance (การเข้าเรียน)
- บันทึกการเข้า-ออก พร้อม timestamp
- เก็บค่า confidence และ method (auto/manual/suggestion)

### Function สำคัญ

#### add_student_embedding
- เพิ่ม embedding ใหม่เข้า array
- ถ้าเกิน 20 จะลบอันเก่าสุดออกอัตโนมัติ
- ใช้ในกระบวนการ rolling update

## การใช้งานระบบ

### ครั้งแรก: ลงทะเบียนนักเรียน

#### ขั้นตอน Setup
1. เปิดระบบและคลิก "ลงทะเบียน"
2. กรอกข้อมูล:
   - ชื่อ-นามสกุล (จำเป็น)
   - ชื่อเล่น (ไม่บังคับ)
3. คลิก "ถัดไป: ถ่ายรูปใบหน้า"

#### การถ่ายรูป (สำคัญมาก!)
**ถ่าย 5 รูป** จากมุมต่างๆ:
1. **รูปที่ 1**: หันหน้าตรง มองกล้อง
2. **รูปที่ 2**: เอียงหน้าซ้ายเล็กน้อย
3. **รูปที่ 3**: เอียงหน้าขวาเล็กน้อย
4. **รูปที่ 4**: ก้มหน้าเล็กน้อย
5. **รูปที่ 5**: เงยหน้าเล็กน้อย

**เคล็ดลับ:**
- ให้แสงสว่างเพียงพอ (แนะนำแสงธรรมชาติ)
- ไม่สวมหน้ากาก
- ไม่ใส่แว่นกันแดด
- อยู่ในกรอบวงกลมสีขาว
- ระยะห่าง 50-100 cm จากกล้อง

### การเช็คชื่อประจำวัน

#### เช็คชื่อเข้า (เช้า)
1. คลิก "เช็คชื่อ"
2. เลือก "เช็คชื่อเข้า (เช้า)"
3. คลิก "สแกนใบหน้า"
4. ให้นักเรียนอยู่ในกรอบ
5. ระบบจะประมวลผลภายใน 1-2 วินาที

**ผลลัพธ์ที่อาจเกิดขึ้น:**

##### 1. ระบบจดจำอัตโนมัติ (Confidence ≥ 85%)
```
✅ บันทึกสำเร็จ!
   ชื่อ: ด.ช. สมชาย ใจดี
   เช็คชื่อเข้า เรียบร้อยแล้ว
```
- บันทึกทันที ไม่ต้องทำอะไรเพิ่ม
- embedding จะถูกเพิ่มเข้า pool อัตโนมัติ

##### 2. แสดงตัวเลือก (Confidence 60-85%)
```
🔍 เลือกนักเรียนที่ถูกต้อง

[ด.ช. สมชาย ใจดี]     78%
[ด.ช. สมศักดิ์ ดีมาก]  72%
[ด.ช. สมหมาย รักดี]    65%

[ไม่มีในรายการ - เลือกเอง]
```
- คลิกเลือกนักเรียนที่ถูกต้อง
- ถ้าไม่มีในรายการ คลิก "เลือกเอง"

##### 3. ให้เลือกเอง (Confidence < 60%)
```
❌ ไม่มั่นใจในการจดจำ
   กรุณาเลือกนักเรียนเอง
```
- จะแสดงรายชื่อทั้งหมด
- เลือกนักเรียนที่ถูกต้อง

#### เช็คชื่อออก (เย็น)
- ทำเหมือนเช็คชื่อเข้า แต่เลือก "เช็คชื่อออก (เย็น)"

### การดู Dashboard

#### Real-time Monitoring
1. คลิก "Dashboard"
2. จะเห็น:
   - จำนวนนักเรียนทั้งหมดที่เช็คชื่อวันนี้
   - จำนวนที่เช็คชื่อเข้าแล้ว
   - จำนวนที่ยังอยู่ในศูนย์ (เข้าแล้วแต่ยังไม่ออก)
   - จำนวนที่เช็คชื่อออกแล้ว

#### รายการแบบ Real-time
- เมื่อมีการเช็คชื่อจากหน่วยบริการอื่น
- Dashboard จะอัพเดตทันที ไม่ต้องกด refresh
- แสดงเวลา เข้า-ออก และวิธีการ (อัตโนมัติ/เลือกเอง)

#### เลือกดูย้อนหลัง
- เลือกวันที่ที่ต้องการดูจาก date picker
- จะแสดงข้อมูลของวันนั้น

## FAQ

### ❓ ทำไมระบบจดจำไม่ได้?

**สาเหตุที่พบบ่อย:**
1. **แสงไม่เพียงพอ** → ถ่ายในที่แสงสว่าง
2. **ระยะไม่เหมาะสม** → ห่างกล้อง 50-100 cm
3. **ใส่หน้ากาก** → ถอดหน้ากากก่อนสแกน
4. **เปลี่ยนแปลงมาก** (ผม, แว่น, หนวด) → ลงทะเบียนใหม่
5. **embeddings น้อยเกินไป** → ต้องมีอย่างน้อย 3 ชุด

**วิธีแก้:**
- ถ้าจดจำไม่ได้บ่อยๆ → ลงทะเบียนใหม่อีกครั้ง
- ระบบจะเรียนรู้ใบหน้าใหม่อัตโนมัติเมื่อเวลาผ่านไป

### ❓ กล้องไม่ทำงาน?

**ตรวจสอบ:**
1. **Permission** - Browser ขออนุญาตใช้กล้องหรือไม่?
   - คลิก 🔒 ข้างๆ URL bar
   - เปิด "Camera" permission
2. **HTTPS** - ต้องใช้ https:// (localhost ใช้ได้)
3. **กล้องถูกใช้งานอยู่** - ปิดแอพอื่นที่ใช้กล้อง

### ❓ ทำไมครั้งแรกช้า?

**เป็นเรื่องปกติ:**
- ครั้งแรกต้องดาวน์โหลด AI models ประมาณ 10-20 MB
- ครั้งถัดไปจะเร็วขึ้น (เก็บใน cache)
- แนะนำให้รอจน "กำลังโหลดระบบจดจำใบหน้า..." หายก่อนใช้งาน

### ❓ Dashboard ไม่ Real-time?

**ตรวจสอบ:**
1. Supabase Realtime เปิดใช้งานหรือไม่?
   - ไปที่ Database → Replication
   - เปิด Realtime สำหรับ table `attendance`
2. เปิด Browser Console (F12)
   - ดู error messages
   - ถ้ามี WebSocket errors → ตรวจสอบ firewall

### ❓ Rolling Update ทำงานอย่างไร?

**กลไก:**
```
Initial Setup:
├── ถ่ายรูป 5 รูป
└── ได้ 5 embeddings

หลังใช้งานไปสักพัก:
├── เช็คชื่อ 15 ครั้ง (confidence ≥ 85%)
├── เพิ่ม 15 embeddings ใหม่
├── รวมเป็น 20 embeddings (5 + 15)
└── เต็มแล้ว!

ครั้งต่อไป:
├── เช็คชื่อสำเร็จอีก 1 ครั้ง
├── เพิ่ม embedding ใหม่ 1 ชุด
├── ลบ embedding เก่าสุด (จาก setup) 1 ชุด ออก
└── คงเหลือ 20 embeddings (ใหม่กว่าเดิม)
```

**ข้อดี:**
- ใบหน้าโต → ระบบเรียนรู้ไปเรื่อยๆ
- ผมยาว/สั้น → ปรับตัวอัตโนมัติ
- ไม่ต้องลงทะเบียนใหม่

### ❓ ข้อมูลปลอดภัยแค่ไหน?

**ความปลอดภัย:**
1. **Face Detection บน Browser** - ไม่ส่งภาพไปยัง Server
2. **เก็บแค่ Embeddings** - ตัวเลข ไม่ใช่รูปภาพ
3. **ไม่สามารถกลับเป็นรูปได้** - Embedding ไม่สามารถ reverse กลับเป็นใบหน้าได้
4. **Supabase Row Level Security** - ป้องกันการเข้าถึงข้อมูลโดยไม่ได้รับอนุญาต

### ❓ ใช้งานออฟไลน์ได้ไหม?

**บางส่วนเท่านั้น:**
- ✅ Face Detection - ทำงานได้ (on-device)
- ✅ หน้า UI - แคชไว้ (PWA)
- ❌ บันทึกข้อมูล - ต้องมี internet (Supabase)
- ❌ Dashboard - ต้องมี internet (Realtime)

**แนะนำ:**
- ควรมี internet connection ขณะใช้งาน

### ❓ รองรับกี่คนต่อระบบ?

**Scalability:**
- นักเรียน: **ไม่จำกัด** (Supabase free tier: 500 MB database)
- การเช็คชื่อต่อวัน: **ไม่จำกัด**
- Concurrent users: ขึ้นกับ Supabase plan

**ตัวอย่าง:**
- 40 คนศูนย์หลัก + 15 หน่วยๆ ละ 15 คน = 265 คน
- ใช้ database ประมาณ 50-100 MB
- ยังอยู่ใน free tier สบายๆ

### ❓ Deploy ไปไหนได้บ้าง?

**แนะนำ:**
1. **Vercel** (ฟรี)
   - Deploy ง่าย
   - รองรับ Next.js โดยตรง
   - Edge functions

2. **Netlify** (ฟรี)
   - Deploy ง่ายเช่นกัน
   - รองรับ Next.js

3. **Self-hosted**
   - VPS (DigitalOcean, Linode)
   - ต้องตั้งค่า Node.js เอง
